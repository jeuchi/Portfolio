Sorting Visualizer (React.js)

I wanted to challenge myself to see if I can build a way to visualize sorting algortihms such as Bubble Sort. By using React.js, I built a dynamic website that randomly generates an array of numbers that are displayed as bars on a graph. The user can select the type of algorithm from the drop down menu and click on Sort. The sorting algorithm will start running and each 'frame' is rendered on the screen, to denote every step of the process. This can be 'comparing 36 and 24' or 'swapping 36 and 24'. The user can stop or reset the sorting at any time. They also have the option to choose the speed of the animation using the slider. This was a fun way for me to really understand these algorithms visually while increasing my skills in Javascript and rendering animations.

HTTP Server and Load Balancer (C Language)

I converted my single threaded HTTP server that handles GET, HEAD, and PUT requests to also manage concurrent requests by using the pthread library with mutex locks. Another requirement was to log each request sent by a client to a file if the user specifies one. Logging means to output the file’s content from the request to a log. The load balancer seperately takes the port number of each multi-threaded server running and does a health check every X seconds or R requests. The client only communicates with the load balancer to redirect them to a healthy server. This uses multi-threading and socket programming to set up client and server connections on each port.

Grocery Store Simulation (C Language)

This project is to simulate a set of jobs performed with a variable number of processors. When there are more jobs than processors, some jobs may have to wait in a queue. Think of shoppers at a grocery store waiting in line at check-out stands. A job is a basket of goods to be purchased and a processor is a clerk at a check-out stand ringing up the purchase. Abstractly, a job is an encapsulation of three quantities: arrival time, duration, and finish time.
